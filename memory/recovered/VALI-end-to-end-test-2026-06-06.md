# VALI — End-to-End Test (Sat, 2026-06-06)

## Mission
Design the end-to-end test that proves the complete order flow works from HTML form submission through webhook validation, payment link generation, notifications, and Sheets logging.

## Context
- HTML form (`order-form.js` v2.0) POSTs JSON to `https://utopia-api.systack.net/webhook/utopia-deli-html-order-v1`
- NEW webhook (`utopia-deli-html-order-v1.json` v1.0.2) processes all-at-once
- Legacy workflow processes per-stage (Contact → Item → Cart → Checkout)
- Fleet Canon adapter is deployed at `utopia-deli-html-order` (NOT `-v1`) and FAILS with missing Square credential
- Our v1 JSON is built but NOT imported into n8n

## Reference: Legacy Per-Stage Flow

```
Contact Form (Google Forms)
    ↓ [triggers webhook]
Item Form (Google Forms)
    ↓ [triggers webhook]
Cart Form (Google Forms)
    ↓ [triggers webhook]
Checkout Form (Google Forms)
    ↓ [triggers webhook]
Order Received (n8n workflow)
    ↓
Square Payment Link
    ↓
Payment Completed / 2 AM Expiration (separate workflows)
```

**Legacy data flow per stage:**
1. **Contact:** name, email, phone → Sheets (contact sheet)
2. **Item:** item selection, quantity, modifiers → Sheets (items sheet)
3. **Cart:** aggregated items, running subtotal → Sheets (cart sheet)
4. **Checkout:** totals confirmation, pickup time → triggers Order Received
5. **Order Received:** creates Square link, emails customer, logs to master sheet

## NEW All-At-Once Flow

```
HTML Form (single page)
    ↓ POST JSON
NEW Webhook (all-at-once)
    ↓
Validate + Normalize Schema
    ↓
Hours Gate
    ↓
Generate Order ID
    ↓
Build Square Line Items
    ↓
Create Square Payment Link
    ↓
Extract Payment Link
    ↓
┌──────────┴──────────┐
Email Customer    Email Kitchen
    ↓                   ↓
    └──────────┬──────────┘
               ↓
        Log to Google Sheets
               ↓
        Respond to Webhook (success/error JSON)
```

## Critical Gaps for End-to-End

### GAP-E2E-001: No Duplicate Order Protection (PESSI-025)
- Double-click submit → two orders, two charges, two kitchen emails
- **Fix:** 5-layer dedup (client token, fingerprint, deterministic key, dedup cache, rate limit)
- **Priority:** HIGH — blocks shipping

### GAP-E2E-002: No Payment Link Lifecycle Management (VALI-009)
- No "Payment Completed" workflow → link stays active after payment
- No "2 AM Expiration" workflow → unused links accumulate
- **Fix:** Build post-payment and expiration workflows
- **Priority:** HIGH — financial/operational risk

### GAP-E2E-003: Form Has No Retry/Recovery on Network Failure
- Customer loses order data on disconnect
- **Fix:** Store form state in localStorage, restore on reconnect
- **Priority:** MEDIUM — UX improvement

### GAP-E2E-004: Sheets Failure = Missing Audit Trail
- If Sheets write fails, order exists in Square but not in master log
- **Fix:** Retry Sheets write with exponential backoff, alert on permanent failure
- **Priority:** MEDIUM — operational risk

## New Pitfall Discovered

**VALI-010: End-to-End Tests Must Verify Every Handoff, Not Just the Webhook**
- **Context:** It's tempting to test the webhook in isolation (post JSON, check response). But the real failure modes happen at handoffs: form→network, network→webhook, webhook→Square, webhook→email, webhook→Sheets.
- **Failure Mode:** Webhook tests pass in isolation, but production fails because the HTML form sends a slightly different field name, or CORS is blocked, or email SMTP is down, or Google API quota exceeded.
- **Fix:** End-to-end tests must trace the entire path: open form → fill fields → submit → verify webhook response → verify email delivery → verify Sheets row → verify Square link → (optional) verify payment completion.
- **Scope:** This is more than a unit test — it's an integration test requiring real infrastructure (or high-fidelity mocks).
