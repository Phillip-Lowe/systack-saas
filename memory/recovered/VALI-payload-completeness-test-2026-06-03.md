# VALI — Payload Completeness Test (Wed, 2026-06-03)

## Mission
Design the test that proves a complete order payload has everything required for the NEW webhook to process it successfully.

## Context
- The NEW webhook (`utopia-deli-html-order-v1.json`) receives a COMPLETE order in one POST
- The HTML form (`order-form.js`) sends integer cents (`subtotal_cents`, `tax_cents`, `total_cents`)
- The webhook's `Validate JSON Structure` node expects float dollars (`subtotal`, `tax`, `total`)
- **PESSI-016 CRITICAL:** Webhook validates fields that don't match form schema → ALL orders fail

## Reference: Legacy Per-Stage Validation

The legacy workflows validate incrementally:

| Stage | Validates | Source |
|-------|-----------|--------|
| Contact form | `customer_name`, `email`, `phone` | User input |
| Item form | `item_id`, `quantity`, `price` (from hardcoded menu) | Hardcoded options |
| Cart renderer | Subtotal = sum(items), totals recalc | Server-side |
| Checkout | `pickup_time`, `special_instructions` | User input |
| Order Received | Final totals check, hours gate | Server-side |

## Summary

**CRITICAL FINDINGS:**
1. **PESSI-016 Schema Mismatch:** The NEW webhook checks `subtotal`/`tax`/`total` (floats) but the form sends `subtotal_cents`/`tax_cents`/`total_cents` (integers). This breaks ALL orders.
2. **PESSI-014 Type Confusion:** `subtotal_cents: 1300` passes `typeof === 'number'` but is treated as $1300.00 instead of $13.00.
3. **Quantity/Price Mismatch:** Webhook checks `item.qty` and `item.price`, form sends `item.quantity` and `item.unit_price_cents`.
4. **Missing Cross-Checks:** No validation that `line_total_cents = quantity * unit_price_cents` or that item prices match a menu catalog.

**NEW PITFALL DISCOVERED:**
- **VALI-006:** `line_total_cents` is sent by form but NEVER validated by webhook. A manipulated `line_total_cents` (e.g., set to 0) would pass all checks if `subtotal_cents` is also manipulated to match. The webhook must independently calculate: `subtotal_cents = Σ(quantity × unit_price_cents)` and reject if client-sent `subtotal_cents` or `line_total_cents` diverges.
