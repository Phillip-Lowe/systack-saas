## Utopia Deli Order System — Production Fixes (04:30-16:10 CDT)

### Bug 1: Sides Not Adding to Cart

**Symptom:** Sandwiches/specialties added to cart but NO sides would add.

**Root Cause 1a — addToCart used stale global state:**
`addToCart(id)` used `selectedModifiers` and `itemQty` global variables populated only when clicking item card. Clicking "Add to Order" directly without selecting first meant stale/wrong data.

**Root Cause 1b — findItem missing MENU.sides:**
```javascript
function findItem(id) {
  return [...MENU.sandwiches, ...MENU.specialties].find(i => i.id === id);  // ❌ NO .sides!
}
```
`findItem('fries-plain')` returned `undefined`, causing `addToCart` to return early.

**Fix:**
- Rewrote `addToCart` to read qty/modifiers from DOM per-item (`qty-${id}`, `mods-${id}`)
- Added `...MENU.sides` to `findItem`

**Deployed:** `index.html` (repo root) — GitHub Pages

### Bug 2: Checkout Form Hang (No Payment Link)

**Symptom:** "Send Payment Link" button showed spinner indefinitely, no error, no email.

**Root Cause 2a — Deployed site ≠ local code:**
The deployed `index.html` had inline JavaScript — NOT the local `order-form.js` that CODY fixed. The fix never reached production.

**Root Cause 2b — DNS misconfiguration:**
`n8n.theutopiadeli.com` CNAME pointed to `fc0bcffc...cfargotunnel.com` which only returned unreachable IPv6. Must use `cloudflared tunnel route dns` to create proper A records.

**Fix:**
- `git reset --hard origin/main` + re-applied fix cleanly
- Deleted broken CNAME, used `cloudflared tunnel route dns n8n-utopia-clean n8n.systack.net`
- Updated `config.js` to use `https://n8n.systack.net/webhook/utopia-deli-html-order-v1`

### Bug 3: n8n Validation Errors

**Symptom:** n8n responded with field validation errors after payload reached it successfully.

**Root Cause 3a — Wrong field names:**
Frontend sent `customer.name`, `items`, `frontend_total_cents` but n8n expected `customer_name`, `order_items`, `subtotal`, `tax`, `total`, `pickup_time`.

**Root Cause 3b — Tax rounding mismatch:**
Frontend sent `Math.round(subtotal * 0.0952)` but n8n expected exact decimal (e.g., `190.40` not `190`).

**Root Cause 3c — Price format mismatch:**
Frontend sent `price` in cents (1300) but n8n expected dollars (13.00) because it does `Math.round(item.price * 100)` for Square API.

**Fix:**
- Updated payload to send n8n-compatible field names
- Changed `tax: subtotal * 0.0952` (exact decimal, not rounded)
- Changed `total: parseFloat((subtotal + tax).toFixed(2))`
- Changed item `price: cartItem.unitPrice / 100` (cents → dollars)

**Key n8n workflow insight:** The `Generate Order ID` node builds Square API body with:
```javascript
line_items: (payload.order_items || []).map(item => ({
  name: item.name,
  quantity: String(item.qty),
  base_price_money: {
    amount: Math.round(item.price * 100),
    currency: "USD"
  }
}))
```
So `item.price` MUST be in dollars (13.00) not cents (1300).

### Bug 4: JavaScript Parse Error Broke Entire Site

**Symptom:** ALL menu items disappeared. Console showed `renderCategory is not defined`.

**Root Cause:** Duplicate `const frontend_total_cents` declaration caused syntax error, preventing ALL inline functions from loading.
```javascript
const frontend_total_cents = subtotal + Math.round(tax);
const frontend_total_cents = subtotal + tax;  // ← DUPLICATE
```

**Fix:** Removed duplicate declaration.
