# 2026-06-05 03:11 CDT — V2 Workflow "Empty" Fix

## Problem
User reported deli workflow V2 showing as "empty" in n8n UI.

## Root Cause
`activeVersionId` was `NULL` in `workflow_entity` table.

n8n UI uses `activeVersionId` to determine which version to display. When it's NULL, the UI shows an empty canvas even though the workflow data exists in the database.

## Diagnosis Steps
1. ✅ Verified workflow exists in database (17,790 bytes of nodes)
2. ✅ Verified nodes are valid JSON (14 nodes, all with positions)
3. ✅ Verified `versionId` matches `publishedVersionId` 
4. ❌ Discovered `activeVersionId` was `NULL`

## Fix Applied

```sql
UPDATE workflow_entity 
SET activeVersionId = 'v2-initial-29ebdb3c',
    meta = '{"templateId": null}'
WHERE id = '29ebdb3c-6dac-4d3a-a119-5cdcc5707e48';
```

Also fixed monitoring workflow with same issue:
```sql
UPDATE workflow_entity 
SET activeVersionId = 'monitoring-v1-initial',
    meta = '{"templateId": null}'
WHERE id = 'a1b2c3d4-1234-5678-9abc-def012345678';
```

## Prevention
Always set `activeVersionId` when deploying workflows via SQL:
```python
activeVersionId = versionId  # Must match!
```

## Status
✅ Fixed. V2 workflow should now display correctly in n8n UI.

**Note:** If UI still shows empty, try hard refresh (Ctrl+Shift+R) or restart n8n.
