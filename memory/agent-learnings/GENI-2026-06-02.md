### Design Philosophy
**"Show, don't just tell."** The customer just completed a transaction. They need:
1. **Confirmation** — yes, we got your order
2. **Verification** — here's what you ordered and how much
3. **Next steps** — here's how to pay and when to pick up
4. **Trust** — consistent with the form they just used

### JavaScript (replaces `showConfirmation`)

```javascript
function showConfirmation(data) {
  // data = { order_id, payment_link, message }
  const main = document.querySelector('.main');
  const hero = document.querySelector('.hero');
  if (main) main.style.display = 'none';
  if (hero) hero.style.display = 'none';
  
  // Populate order details
  document.getElementById('confirmOrderId').textContent = data.order_id;
  document.getElementById('confirmPickupTime').textContent = data.pickup_time || 'ASAP';
  document.getElementById('confirmCustomer').textContent = data.customer_name;
  
  // Populate items
  const itemsContainer = document.getElementById('confirmItems');
  itemsContainer.innerHTML = data.order_items.map(item => `
    <div class="confirmation-item">
      <div>
        <div class="confirmation-item-name">${item.qty} × ${item.name}</div>
        ${item.modifiers?.length ? `
          <div class="confirmation-item-modifiers">${item.modifiers.join(' • ')}</div>
        ` : ''}
      </div>
      <div class="confirmation-item-price">$${(item.price * item.qty).toFixed(2)}</div>
    </div>
  `).join('');
  
  // Populate totals
  document.getElementById('confirmSubtotal').textContent = `$${data.subtotal.toFixed(2)}`;
  document.getElementById('confirmTax').textContent = `$${data.tax.toFixed(2)}`;
  document.getElementById('confirmTotal').textContent = `$${data.total.toFixed(2)}`;
  
  // Set payment link
  const payLink = document.getElementById('confirmPayLink');
  if (data.payment_link) {
    payLink.href = data.payment_link;
    payLink.style.display = 'inline-block';
  } else {
    payLink.style.display = 'none';
  }
  
  // Show confirmation
  document.getElementById('confirmationPage').style.display = 'flex';
  window.scrollTo(0, 0);
}
```

## Design Decisions

1. **Gradient header matching the form** — visual continuity from form → confirmation
2. **Gold badge for order ID** — makes it scannable and referenceable
3. **Item-level detail** — qty, name, modifiers, line total — customer can verify
4. **CSS variables** — uses the same design tokens as the form, stays consistent
5. **Hidden in HTML, shown via JS** — no DOM injection, graceful degradation possible
6. **Mobile-first** — max-width 600px, padding scales, flex layouts
7. **Pill CTA** — matches the form's "Send Payment Link" button style
8. **Expiration urgency** — "Links expire at 2:00 AM CT" drives action
9. **Phone number** — clickable tel: link for immediate help

---

## NEW Pitfall Discovered

**GENI-006: Inline confirmation pages hide critical information from the customer.**

The current `showConfirmation()` creates a generic "We Got You!" message with NO order details. After submitting an order, the customer needs to see WHAT they ordered, for HOW MUCH, and WHEN to pick it up. Without this, they feel anxious and may call to verify — defeating the purpose of automation.

**Fix:** Confirmation page must display: order ID, pickup time, item list with modifiers, subtotal, tax, total, and payment CTA. Use the same CSS variables as the form for visual consistency.
